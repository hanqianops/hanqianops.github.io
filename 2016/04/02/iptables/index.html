iptables应用

  • secure

Linux防火墙主要工作在网络网络层，针对TCP/IP数据包过滤和限制
netfilter（内核态）：指的是Linux内核中实现包过滤防火墙的内部结构，不以程序和文件的形式存在
iptables（用户态）：指用来管理Linux防火墙的命令程序，位于/sbin/iptables
以上都是用来指Linux防火墙


iptables结构

4个规则表

4个表的应用顺序：raw-->mangle-->nat-->filter

  • raw 用来决定是否对数据包进行状态跟踪
    对应模块iptable_raw；
    包含两个链OUTPUT、PREROUTING
  • mangle 改变数据包的内容
    对应模块iptable_mangle；
包含五个链
  • nat 网络地址转换
    对应模块iptable_nat
包含三个链OUTPUT、PREROUTING、POSTROUTING
  • filter 过滤数据包
    对应内核模块iptable_filter
包含三个链INPUT、OUTPUT、FORWARD

5种规则链

  • INPUT（入站）：收到访问防火墙本机地址的数据包时应用
  • OUTPUT（出站）：本机发送数据包时应用
  • PREROUTING（路由前）：在对数据包做选择路由之前应用
  • FORWARD（转发）：当接收到需要通过本机转发给其它地址的数据包时应用
  • POSTROUTING（路由后）：在对数据包做完选择路由之后应用

规则链应用顺序

  • 入站数据：PREROUTING –> INPUT
  • 出站数据：OUTPUT –> POSTROUTING
  • 转发数据：PREROUTING –> FORWARD –> POSTROUTING

控制类型

  • DROP：悄悄丢弃；一般我们多用DROP来隐藏我们的身份，以及隐藏我们的链表
  • REJECT：明示拒绝；显示无法连接到端口
  • ACCEPT：接受
  • LOG：记录日志到/var/log/messages;要写到最前边

基本语法

  • iptable [-t 表名] [-(A|I|D|R)链名] [规则编号] [-(i|o）eth0] [-p 协议类型] [-s 源地址/子网 ] [–sport 源端口] [-d 目的地址] [–dport 目的端口] [-j 控制类型]

命令参数

  • -A 在链的末尾追加一条规则
  • -I 在链的开头（或指定序号）插入一条规则
  • -R 修改、替换，指定规则序号
  • -L 列出所有的规则条目
  • -n 以数字形式显示地址、端口等信息
  • -v 以更详细的方式显示规则信息
  • –line-numbers 查询规则时，显示规则的序号
  • -D 删除链内指定序号（或内容）的一条规则
  • -F 清空所有的规则，默认策略不受影响
  • -Z 清空 “-vn”显示的数据包
  • -P 为指定的链设置默认规则

匹配类型

  • 匹配协议
    -p 协议名
！-p 协议名  #取反
  • 匹配地址
    -s 源地址
-d 目的地址
  • 匹配接口
    -i 入站网卡
-o 出站网卡
  • 匹配端口
    --sport 源端口
--dport 目的端口
  • 匹配TCP标记 
    --tcp-flags 检查范围 被设置的标记
  • 匹配ICMP
    --icmp-type ICMP类型
  • 匹配多端口
    -m multiport --sports | --dports 端口列表
  • 匹配IP范围
    -m iprange --src-range IP范围
  • 匹配MAC地址
    -m mac --mac-source MAC地址
  • 匹配状态
    -m state --state 连接状态

自定义链

  • iptables -N aaa 创建自定义链
  • iptables -A INPUT -j aaa 引用到INPUT链
  • iptables -X 清除自定义链
