top实时分析系统性能

  • 系统

top是一个动态显示过程,即可以通过用户按键来不断刷新当前状态.如果在前台执行该命令,它将独占前台,直到用户终止该程序为止。top命令提供了实时的对系统处理器的状态监视。


输出如下：

第一行

  • 17:12:04 当前系统时间
  • 1 day, 2:45   系统已经持续运行的时间
  • 4 users     当前有4个用户登录系统
  • load average: 0.00, 0.01, 0.00   1分钟、5分钟、15分钟的负载情况。

第二行

  • 92 total     系统现在有92个进程
  • 1 running     有1个正在运行
  • 90 sleeping    90个在休眠状态
  • 1 stopped     有一个进程在停止状态
  • 0 zombie     系统没有僵尸进程

第三行

  • 0.5%us     用户占用CPU百分比
  • 0.2%sy     系统占用CPU百分比
  • 0.0%ni     改变过优先级的进程占用CPU的百分比
  • 99.2%id     空闲CPU百分比
  • 0.0% wa     IO等待占用CPU的百分比
  • 0.0% hi     硬中断（Hardware IRQ）占用CPU的百分比
  • 0.0% si     软中断（Software Interrupts）占用CPU的百分比

第四行

  • 内存情况：总内存、已使用内存、剩余内存、缓冲使用内存。

第五行

  • 交换分区情况： swap总大小、已使用大小、剩余大小、缓存使用大小。

第六行

  • PID     进程id
  • USER      进程所有者
  • PR      进程优先级
  • NI      nice值。负值表示高优先级，正值表示低优先级
  • VIRT     进程使用的虚拟内存总量，单位kb。VIRT=SWAP+RES
  • RES      进程使用的、未被换出的物理内存大小，单位kb。
  • RES     RES=CODE+DATA
  • SHR     共享内存大小，单位kb。
  • S      进程状态。D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
  • %CPU     上次更新到现在的CPU时间占用百分比
  • %MEM      进程使用的物理内存百分比
  • TIME+      进程使用的CPU时间总计，单位1/100秒
  • COMMAND     进程名称

第七行

  • 运行的进程

视图中的操作

  • 按1：显示CPU逻辑核数
  • 按P：按CPU大小排序
  • 按M：按内存大小排序
  • 按k：在视图中杀进程
  • 按c：显示进程的完整命令
  • 按b：高亮显示
  • 按f：进入另一个视图，可以选择显示的字段
