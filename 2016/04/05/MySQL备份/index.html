MySQL备份

  • mysql

备份的作用：
迁移或升级数据库时
增加从库时
因硬件或特殊异常情况，（主库或从库宕机，主从可以互相切换，无需备份）
认为的DDL、DML语句，主从没办法，所有库都会执行。此时需要备份
跨机房灾备，需要备份到异地


#mysqldump

  • -A 备份全部数据库 –all-databases
  • -B 指定多个库，增加建库语句和use语句 –database
  • -d 只备份表结构
  • -t 只备份表数据
  • -F 备份后刷新binlog，备份多个库刷新多次 –flush-logs
  • -x 锁表(MyISAM使用) –lock-all-tables
  • –default-character-set=latin1 指定备份字符集
  • –compact 去注释，适合调试，生产不用
  • –triggers 备份触发器。默认启用，用–skip-triggers禁用它。
  • -R意思是备份存储过程和函数

备份表

  • mysqldump -u root -p 库名 表名> mysql.sql

备份库

  • mysqldump -u root -p -B 库名 > mysql.sql

恢复数据

  • mysql -u root -p 库名< mysql.sql
    恢复表需要加库名，恢复库什么都不用加

通过SSH将MySQL数据库复制到新服务器

mysqldump –add-drop-table –extended-insert –force –log-error=error.log -uUSER -pPASS OLD_DB_NAME | ssh -C user@newhost “mysql -uUSER -pPASS NEW_DB_NAME”
通过压缩的SSH隧道Dump一个MySQL数据库，将其作为输入传递给mysql命令，我认为这是迁移数据库到新服务器最快最好的方法。
